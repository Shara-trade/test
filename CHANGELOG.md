# История изменений

Все заметные изменения проекта документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

## [1.0.0] - 2025-01-14

### Added

#### Архитектура
- Единая структура проекта с разделением на handlers, game, core, database, admin
- Система миграций базы данных
- Connection pool для оптимизации запросов
- Кэширование часто используемых данных

#### Безопасность
- Переменные окружения через `.env` (python-dotenv)
- Подпись callback_data (HMAC-SHA256)
- Защита от спама (rate limiter)
- Защита от быстрых кликов (click protector)
- Проверка владения ресурсами (ownership middleware)
- Декоратор `@admin_required()` для проверки прав

#### База данных
- Полная схема БД с индексами
- Внешние ключи с CASCADE
- Оптимизированные запросы с подзапросами
- Методы для работы с пользователями, дронами, инвентарём, модулями
- Кэширование бонусов модулей (TTL 10 сек)

#### Игровые механики
- **Добыча**: кликовая система с астероидами
- **Уровни**: полиномиальная формула опыта
- **Перегрев**: система риска и награды
- **Дроны**: 5 типов с пассивным доходом и синергией
- **Модули**: генерация с бафами/дебафами, 5 редкостей
- **Контейнеры**: 5 типов редкости + КСМ
- **Материалы**: 15 материалов в 3 группах
- **Крафт**: система рецептов
- **Кланы**: корпорации игроков
- **Экспедиции**: отправка дронов

#### Хендлеры
- Унифицированная обработка callback (BaseCallbackHandler)
- Валидация ввода (InputValidator)
- Пагинация списков (Paginator)
- Топы с реальными данными из БД

#### Админ-панель
- Система ролей (owner, senior, moderator, support)
- Гибкие права доступа
- Управление игроками (бан, редактирование ресурсов)
- Настройки бота (шансы, множители, лимиты)
- Логирование действий админов

#### Тестирование
- Unit-тесты для игровых механик
- Тесты базы данных
- Тесты валидаторов
- Фикстуры pytest

#### Документация
- README.md с инструкцией по установке
- CHANGELOG.md с историей изменений
- Документация в коде (docstrings)

## Типы изменений

- `Added` — новый функционал
- `Changed` — изменения в существующем функционале
- `Deprecated` — функционал, который будет удалён
- `Removed` — удалённый функционал
- `Fixed` — исправленные баги
- `Security` — исправления уязвимостей
